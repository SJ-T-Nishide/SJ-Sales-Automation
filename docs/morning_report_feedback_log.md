#### 1. 経営コンテキストに追加すべき内容
```markdown
【営業トークと実態の区別】
```

- 「顧客面談の内容は営業トークの可能性も考慮して重要度を判定する」と明記

## 改善内容の実装 #1（2026-05-23）

### プロンプトに追加した内容

**1. 営業トークと実態の区別セクション**
```markdown
顧客面談での撤退・損失・市況悪化の話は、営業トークの一部である場合がある。
一概に最重要シグナルとはしない。

実際の経営判断に必要なのは：
- 新規顧客獲得施策の進捗・期限
- 営業担当の顧客対応の質
- 市況変化の構造的な兆候（実数データで確認）
```

**2. 重要度判定の明確な基準**
- **Sランク:** 期限が明確（1週間以内）、新規施策の進捗、実数データあり
- **Aランク:** 営業トークの内容分析、市況変化の兆候
- **B/Cランク:** 個別顧客の撤退話、実数データなしの推測

**3. 実数データチェック**
- 具体的な数字がない場合は重要度を1段階下げる
- 「約〇〇」「〇〇の可能性」は営業トークとして扱う

### 期待される効果
- 営業トークを過大評価しなくなる
- 期限付きタスク・施策が適切にSランクになる
- 実数データの有無で重要度が調整される
