# 手動朝レポート生成プロンプト

> **Phase 2（手動テスト期間）専用**  
> 毎朝このプロンプトを使って、前日の情報をClaudeに投げて朝レポートを生成します。

---

## 使い方

1. 前日のPlaud AI文字起こし、tl;dv要約、Slackログ、会議メモを収集
2. 下記のプロンプトをコピー
3. `【入力データ】` の部分に収集した情報を貼り付け
4. Claude（Web版 or Claude Code）に投げる
5. 生成されたレポートを確認
6. [フィードバックテンプレート](./04_feedback_template.md)でフィードバックを記録

---

## プロンプトテンプレート

```
以下の情報をもとに、経営者向けの「朝レポート」を作成してください。

目的は、昨日社内で起きた重要な動き・未対応タスク・社長判断事項を
経営判断に役立つ形で把握することです。単なる要約ではなく、
「何が重要か」「何をすべきか」が分かる形にしてください。

---
【経営コンテキスト】

当社は、民泊経営パッケージを中心に、民泊・不動産・運営代行・投資家向けパッケージ事業を展開している。

現在AIに優先して拾ってほしい最重要テーマは以下の3つ。

1. 民泊経営パッケージの成約・粗利
重要視するもの：
販売件数、成約件数、成約金額、利益、粗利、有効商談、投資家の温度感、
失注理由、紹介・送客、関連商材の販売可能性。

2. 民泊運営物件の売上・レビュー・利益率
重要視するもの：
売上UP、レビュースコア、清掃クオリティ、クレーム、利益率、経費削減、
オペレーション改善、ゲスト満足度。

3. AI活用による業務自動化・属人性排除
重要視するもの：
手作業削減、業務の正確性向上、属人業務の排除、マニュアル化、
自動化可能な業務、繰り返し発生している確認作業、AI利用による生産性向上。

部門別の見るポイント

営業部門：
民泊経営パッケージの販売件数、成約件数、金額、利益を最重視する。
加えて、民泊を題材とした他商材の開発・販売可能性、理念浸透、
AI利用による営業業務の自動化も重視する。

管理部門：
民泊運営のレビュースコアアップ、売上UP、清掃クオリティアップ、
利益率UP、経費削減を重視する。
加えて、理念浸透、AI利用による運営業務の自動化も重視する。

コーポレート部門：
AI利用による業務自動化、正確性向上、属人性の排除を重視する。
加えて、理念浸透、AI利用によるバックオフィス業務の自動化も重視する。

経営陣：
売上成長率、営業利益率、新規事業、理念浸透、AI利用による全社業務の自動化を重視する。

理念浸透について
理念浸透は全社共通の横串テーマとして扱う。
理念に沿った良い行動、理念とズレている行動、顧客・投資家・ゲストへの対応に
理念が出ている場面、社内で判断基準がズレている場面を重要シグナルとして拾う。

AIに拾ってほしいシグナル
- 民泊経営パッケージの成約に近い話
- 投資家の温度感が上がった／下がった話
- 成約・失注・停滞の兆候
- 新商材になりそうなアイデア
- 運営物件の売上UPにつながる話
- レビュー低下やクレームにつながる兆候
- 清掃品質や現場品質の問題
- 利益率を上げる改善案
- 経費削減につながる話
- 属人化している業務
- AIやGASで自動化できそうな業務
- 契約・請求・支払・法務・経理のミスにつながる兆候
- 社長・幹部判断が必要な事項
- 理念に沿った良い行動 / 理念からズレている行動

AIに無視してよいノイズ
- 単なる挨拶
- 雑談
- 重要度の低い日程調整
- 完了済みの軽い報告
- アクションのない感想
- 売上・利益・品質・理念・AI自動化に関係しない話

【出力形式】

1. 今日の総括
昨日の社内の動きを経営者向けに3〜5行でまとめる。

2. 重要シグナル TOP5
重要度が高い順に5つ。各項目は以下の形式で。
- 重要度：S / A / B / C
- 種別：成約/投資家/物件/運営/社内/トラブル/アイデア/自動化/その他
- 内容：
- なぜ重要か：
- 次のアクション：
- 放置リスク：

3. 未対応TODO
会話・会議から発生したTODOを抽出。
- タスク：
- 担当者：（不明な場合は「不明」）
- 期限：（不明な場合は「不明」）
- 重要度：
- 元になった発言：
```

## 出力例イメージ

生成される朝レポートは以下のような形式になります：

```markdown
# 朝レポート 2026-05-20

## 1. 今日の総括
昨日は投資家B社との商談が進展し、3件の追加案件の可能性が浮上。
一方、物件Xのレビューが4.8→4.5に低下、清掃品質の問題が原因。
営業チームから契約書作成の自動化提案あり。

## 2. 重要シグナル TOP5

### #1
- 重要度：S
- 種別：投資家
- 内容：B社が3件の追加案件を検討中。今週中に具体的な提案が必要。
- なぜ重要か：成約すれば粗利+500万円の見込み。
- 次のアクション：営業部が今週中に提案書を作成。社長が最終レビュー。
- 放置リスク：他社に流れる可能性。今週中に動かないと失注の恐れ。

### #2
- 重要度：S
- 種別：運営
- 内容：物件Xのレビューが4.8→4.5に低下。ゲストコメントで清掃品質を指摘。
- なぜ重要か：レビュー低下は売上に直結。今後の予約にも影響。
- 次のアクション：清掃業者Zに即座にヒアリング。品質チェック体制を見直し。
- 放置リスク：さらにレビューが下がり、物件Xの売上が-20%になる可能性。

### #3
- 重要度：A
- 種別：自動化
- 内容：営業チームから「契約書作成を毎回手作業でやっている。GASで自動化できないか」との提案。
- なぜ重要か：契約書作成に1件あたり30分かかっている。月20件で10時間削減可能。
- 次のアクション：コーポレート部門でGAS自動化の検討。来週までに試作版を作成。
- 放置リスク：属人化が続き、ミスリスクが高まる。

（以下略）

## 3. 未対応TODO

- タスク：B社向け提案書作成
- 担当者：営業部A
- 期限：2026-05-23
- 重要度：S
- 元になった発言：「B社から3件追加の話が出たので、今週中に提案書を作ります」

- タスク：物件Xの清掃業者ヒアリング
- 担当者：管理部B
- 期限：2026-05-21
- 重要度：S
- 元になった発言：「レビューが下がってるので、明日清掃業者に確認します」

（以下略）
```

## 注意事項

### 最初の1週間は軽く
- 最初の1週間はセクション3つ（総括・シグナルTOP5・TODO）に絞る
- 2週目以降に項目を追加する（理念浸透スコア、週次トレンドなど）

### フィードバックが命
- 毎日必ず[フィードバックテンプレート](./04_feedback_template.md)でフィードバックを記録
- 金曜日に1週間分のフィードバックをまとめてClaudeに投げる
- 「経営コンテキストの更新案を出して」と依頼して、判断基準を進化させる

### プライバシー配慮
- 個人情報・機密情報が含まれる場合は、手動で削除またはマスキングしてから投げる
- 録音データは業務目的のみ・私的会話は対象外
